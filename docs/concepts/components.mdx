# Component Types

## Abstract Components

ie ModelFilter

Selects a more specific component based on model and attribute data.

## Composite Components

ie FilterGroup, FilterLayoutVertical

Groups multiple components together.

## Convenience Components

ie FieldGroupPassword

Use `useMergedOverrides` to merge the overrides for the composed component with the overrides for the convenience component.

Same overrides API as FieldGroup

## Base Components

ie FieldInput

A component that forms the base of more specific components.
